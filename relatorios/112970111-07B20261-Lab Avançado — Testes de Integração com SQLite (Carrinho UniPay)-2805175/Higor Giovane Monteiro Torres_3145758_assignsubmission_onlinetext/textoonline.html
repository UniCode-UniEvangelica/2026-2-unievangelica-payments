O isolamento acontece porque o identificador :memory: cria um banco de dados que existe apenas na memória RAM de uma conexão específica e desaparece instantaneamente quando ela é fechada. Graças à sua fixture, o Pytest abre e fecha uma conexão nova para cada função de teste, garantindo que os dados de um teste nunca "vazem" para o próximo, pois cada um ganha um banco totalmente virgem. Enquanto no test_pagamentos.py você usa Mocks para "fingir" um comportamento e evitar chamadas externas lentas (como uma API), no teste de carrinho você usa o banco real para validar se o seu SQL está funcionando de verdade, mas com a vantagem de não precisar criar ou apagar arquivos físicos no disco, unindo a confiança de um teste de integração com a velocidade de um teste unitário.
